Урок 1. Початок розробки на C#. Установка .NET

Час виконання: до 20 хвилин

Для компіляції та запуску програм на C# Вам знадобиться .NET SDK, його можна завантажити за посиланням: https://dotnet.microsoft.com/en-us/download

Завантажте та встановіть .NET SDK. Після установки, напишіть в командному рядку dotnet. Якщо установка пройшла успішно, то отримаєте щось на зразок

Usage: dotnet [options]
Usage: dotnet [path-to-application]

Options:
  -h|--help         Display help.
  --info            Display .NET information.
  --list-sdks       Display the installed SDKs.
  --list-runtimes   Display the installed runtimes.

path-to-application:
  The path to an application .dll file to execute.

В терміналі, перейдіть до нової директорії, де Ви бажаєте розмістити новий проект. Створіть перший проект командою dotnet new console

Ви автоматично отримаєте файл проекту та файл з вихідним кодом

using System;

namespace test
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("Hello World!");
        }
    }
}

В даному коді є метод static void Main, який є вхідною точкою для виконання програми. Тобто, яку програму Ви б не писали, завжди виконання починається з методу static void Main. Цей метод може бути поміщений у будь-який клас. Якщо такого методу не буде в жодному з класів програми, то виникне помилка компіляції.

В даному коді метод Main містить лише одну інструкцію.

Console.WriteLine("Hello World!");

Вона означає, що треба вивести на консоль текст "Hello World!". Оскільки в методі Main більше немає ніяких інструкцій, виконання програми закінчується.

Щоб скомпілювати та запустити програму, напишіть команду dotnet run. Отримаєте такий результат:

>dotnet run
Hello wolrd!

Ще один нюанс: усі методи в мові C# мають бути розміщені всередині класів. А от використання просторів імен (namespace) не є обов'язковим. Більш детально про це – у наступних уроках.

Завдання 1: Змініть код програми так, щоб виводилося ваше прізвище та ім'я.

Завдання 2: За допомогою команди dotnet help дізнайтеся про інші можливості .NET SDK.
